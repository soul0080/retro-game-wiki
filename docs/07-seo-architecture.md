# Retro Game Wiki SEO 架构设计文档

版本：v1.0

项目目标：

通过结构化游戏知识库、页面集群、内部链接和搜索优化，建立中文经典游戏领域长期自然流量入口。

---

# 1. SEO总体战略

## 核心目标

打造：

> 中文经典游戏领域的结构化内容搜索引擎。

不是：

简单文章网站。

而是：

```
游戏平台

↓

游戏

↓

攻略

↓

角色

↓

道具

↓

Boss

↓

专题
```

形成完整知识网络。

---

# 2. 搜索引擎目标

主要：

## 国内

* 百度

## 海外

* Google
* Bing

技术标准：

优先按照 Google SEO 标准设计。

# 4. URL设计规范

原则：

## 简短

## 可读

## 稳定

---

# 4.1 首页

URL:

```
/
```

Title:

```
经典游戏百科 - 怀旧游戏攻略与模拟器资料库
```

---

# 4.2 游戏列表

URL:

```
/games
```

用途：

游戏索引页。

---

# 4.3 游戏详情页

URL:

```
/games/{slug}
```

示例：

```
/games/final-fantasy-6
```

SEO价值：

最高。

---

页面关键词：

```
最终幻想6

最终幻想6攻略

FF6资料

FF6角色
```

---

# 4.4 攻略页面

URL:

```
/guides/{slug}
```

示例：

```
/guides/final-fantasy-6-world-of-ruin
```

---

# 4.5 平台页面

URL:

```
/platform/{slug}
```

例如：

```
/platform/gba
```

关键词：

```
GBA游戏推荐

GBA经典游戏
```

---

# 4.6 专题页面

URL:

```
/topics/{slug}
```

例如：

```
/topics/best-sfc-rpg
```

作用：

获取泛关键词。

---

# 5.1 游戏页面模板

这是网站SEO核心。

---

# 6.1 游戏页连接

```

Boss
```

# 18. SEO风险控制

禁止：

## 大量重复模板

解决：

每个游戏：

加入：

* 历史背景
* 游戏特色
* 玩家评价
* 独有资料

---

## AI垃圾内容

必须：

人工审核。

---

## 关键词堆积

禁止：

重复大量关键词。

SEO不是后期优化。

SEO就是网站架构。

本项目核心竞争力：

不是新闻速度。

而是：

```
结构化游戏数据库

+

大量长尾关键词页面

+

长期内容积累
```
